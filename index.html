Agentic orchestration on AWS (Coordinator ↔ Specialists) Event-driven fan-out, bounded state, deterministic retries
SQS Redis Idempotency

Problem

  • Multi-step agent workflows became slow and unpredictable as the number of tools and sub-tasks increased.
  • Needed scalable coordination with controlled state and clear failure handling.

Approach

  • Coordinator issues tasks to specialist agents via SQS; each handler is idempotent and retry-safe.
  • Redis stores session state with TTL and explicit schemas to prevent drift.
  • DLQs capture poison messages; backpressure via queue depth + rate limits.

Reliability defaults

  • Idempotency keys per step
  • Timeouts + retries with jitter
  • Deterministic replay hooks (audit logs)

Results

  • Reduced inter-service latency by ~40% (reported impact).
  • Improved system behavior under load using retries/backpressure and bounded memory.

Vector Search Rerank

Problem

  • LLM answers were inconsistent when key documents were missed or irrelevant passages were retrieved.
  • Needed higher precision while maintaining latency constraints.

Approach

  • Hybrid retrieval: vector recall + keyword precision.
  • Reranking to prioritize evidence; citation discipline to prevent unsupported claims.
  • Evaluation: track error buckets (missed retrieval vs. bad ranking vs. synthesis).

Metrics

  • Precision/recall style checks on curated queries
  • Latency budgets (P95)
  • Regression detection via golden sets

Results

  • Improved retrieval accuracy by ~30% (reported impact).
  • More stable answers due to better evidence selection and citations.

Scaling daily call volume (50K → 300K) Performance tuning without adding infra cost
DB Batching Queues

Problem

  • Call/SMS workloads spiked under peak loads, causing latency and throughput issues.

Approach

  • Batch processing for requests to reduce per-call overhead.
  • DB optimizations: indexing and connection pooling.
  • Horizontal scaling using task queues (e.g., Celery/RabbitMQ pattern).

Results

  • Reduced latency by ~30% and scaled from ~50K to ~300K daily calls.
  • Maintained cost by improving efficiency rather than adding capacity.

Next steps

  • Load-test harness + capacity planning models
  • Queue depth SLOs + autoscaling policies
